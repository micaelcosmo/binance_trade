Binance Trade Bot Pro 🤖 AI Edition

Terminal quantitativo automatizado com análise tridimensional profunda (Contexto 12h), trailing stop e gestão institucional em lote validada pelo Google Gemini.

v3.4.2 Python 3.9+ Tkinter UI Gemini 2.5 Flash-Lite

🧠 Visão Tridimensional Estendida

O motor avalia o mercado fornecendo à IA o histórico completo de Price Action das últimas 12 horas (Macro), além da EMA curta (1H) e do Gatilho MACD (15m). O sistema evita matematicamente a compra de facas caindo.

📈 Risco/Retorno Elástico

Stop Loss dinâmico calculado via ATR (Average True Range) blindado de forma elástica até -7.00% (Altcoins) e -10.50% (BTC). O Trailing Stop persegue alvos invisíveis de +1.50%, suportando a volatilidade do mercado.

🔄 Auto-Updater Seguro

Sincronize o código-fonte direto do GitHub com 1 clique no Painel UI. O sistema possui trava inteligente: aguarda o encerramento de operações em aberto para executar atualizações sem colocar seu capital em risco.

🚨 Manual Override & Cooldown

Assuma o controle via Botão de Venda Forçada. Em cenários complexos, a IA veta mercados laterais (30min cooldown) ou julga matematicamente se vale a pena migrar de uma operação ruim (Tribunal de Swap).

Bem-vindo ao Bot

Este manual guiará você desde a instalação até a sua primeira operação automatizada.

1. Instalação e Preparação

Clone o repositório, crie um ambiente virtual e instale as dependências. O projeto utiliza o SDK oficial google-genai nativo para alta performance.

git clone https://github.com/micaelcosmo/binance_trade.git
cd binance_trade
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt

2. Configurando suas Credenciais

Aviso de Segurança: Nunca ative a permissão de "Saques" (Withdrawals) na sua API da Binance. O bot precisa apenas de permissão de Leitura e Spot Trading.

Faça cópias dos arquivos de exemplo fornecidos na raiz do projeto:

  • Copie user.cfg.example para user.cfg
  • Copie supported_coin_list.example para supported_coin_list.txt

Abra o user.cfg com um editor de texto e insira suas chaves da Binance e a sua GOOGLE_API_KEY (Gerada gratuitamente no Google AI Studio).

3. Lista de Moedas (Watchlist)

Abra o arquivo supported_coin_list.txt. Adicione o símbolo das moedas que você deseja que o bot monitore (uma por linha), sem o par USDT. Exemplo:

BTC
SOL
LINK
PEPE

4. Iniciando o Painel UI e Atualizações

Inicie o Dashboard gráfico executando o seguinte comando no terminal:

python painel.py

No painel, clique em RUN > Iniciar Bot. Você tem à disposição o placar em tempo real, Dossiê de Inteligência e ferramentas de intervenção manual.

Para se manter atualizado com novas versões de estratégia, basta clicar no botão 🔄 Atualizar Versão na interface. O bot fará o download e reiniciará sozinho com segurança.

5. Troubleshooting Comum

  • Erro -1021 (Timestamp outside recvWindow): Relógio do Windows atrasado. Vá em configurações de Data/Hora e clique em "Sincronizar Agora".
  • Erro de Conflito websockets: Em sistemas antigos, pode ser necessário rodar pip install websockets==11.0.3 para forçar o downgrade exigido pela API da Binance.

Arquitetura do Sistema (v3.4.2)

A arquitetura evoluiu para um ecossistema Híbrido padronizado (Inglês no código/Variáveis e Português em Docstrings/UI): Cálculo Local via Dataframes (Pandas-TA) + Auditoria em Nuvem Generativa (Google Gemini).

1. Estrutura Quantitativa (profit_gain_strategy.py)

O motor (Strategy) centraliza a extração assíncrona dos candlests (OHLCV) e calcula a árvore de decisão através do método get_enriched_data:

  • Contexto 12h (Macro): Injeta o histórico contínuo das últimas 12 horas (Price Action) e afere a compressão do ativo via EMA 9 (Reversão de Curto Prazo).
  • Swing de Exaustão (1H): Mede o distanciamento da EMA 21, o slope (inclinação) direcional do RSI e exige que o Histograma MACD demonstre sinais de reversão (`macd_1h_shifting_up`). Calcula o True Bottom exigindo uma queda maior que 2x o ATR.
  • Gatilho Micro (15m): Cruza o MACD Histogram positivo com anomalias de volume institucional no time-frame de aceleração.

2. Agente Institucional: MarketAnalyzer (ai_agent.py)

Integrado via SDK nativo com gemini-2.5-flash-lite. A IA processa o *Chain of Thought* recebendo o payload JSON sob leis estritas:

  • A Lei da Estrutura: Veta matematicamente ativos onde as últimas 12 horas demonstrem colapso descendente contínuo, independente de picos isolados.
  • Trava de Confiança Micro: Impossibilita emissão de *scores* de aprovação (>= 90) sem que o Python confirme *Spikes* de volume acima da média de 10 períodos no 15m.

3. Auto-Updater & IPC (Inter-Process Communication)

A separação entre a Interface (Tkinter/Main Thread) e o Motor Quantitativo (Sub-processo Independente via `subprocess.Popen`) exige um IPC robusto (Non-blocking):

  • Safe Update Cycle: O componente perform_update no painel aciona a sincronização remota (`git reset` e `git pull`). Utiliza a leitura de diretório absoluto (`os.path.abspath`) para o recarregamento na memória via substituição de Popen (`sys.execl` deprecado), garantindo imunidade a *paths* corrompidos de atalhos do Windows.
  • Ghost Flags: Ações de UI, como botões de Venda Forçada ou bloqueio para atualização, geram arquivos .flag que são interceptados silenciosamente pelo motor durante sua varredura (ex: método _check_ui_flags).
