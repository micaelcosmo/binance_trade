Binance Trade Bot Pro 🤖 AI Edition

Terminal quantitativo automatizado com análise tridimensional (MACD/ATR), trailing stop e gestão institucional em lote validada pelo Google Gemini.

v3.3.0 Python 3.9+ Tkinter UI Gemini 2.5 Flash-Lite

🧠 Visão Tridimensional (IA)

O motor avalia o mercado em 4H (Macro), 1H (Elástico EMA) e 15m (Gatilho MACD). O Dossiê é submetido à IA, que garante a formação da "barriga" da reversão antes de comprar.

📈 Risco/Retorno Ajustado

Stop Loss dinâmico calculado via ATR (Average True Range) blindado em no máximo -3.50%. O Trailing Stop persegue alvos de +1.50%, mantendo a matemática a seu favor.

🚨 Manual Override

O Painel UI conta com um Botão de Venda Forçada. Assuma o controle total da operação em caso de notícias macroeconômicas, registrando o P/L de forma perfeitamente assíncrona.

👑 Auto-Switch & Cooldown

A IA veta mercados laterais poupando tokens (30min cooldown). Caso esteja preso num trade, o Tribunal de Swap julga matematicamente se vale a pena migrar o capital.

Bem-vindo ao Bot

Este manual guiará você desde a instalação até a sua primeira operação automatizada.

1. Instalação e Preparação

Clone o repositório, crie um ambiente virtual e instale as dependências. O projeto utiliza google-genai nativo para alta performance.

git clone https://github.com/micaelcosmo/binance_trade.git
cd binance_trade
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt

2. Configurando suas Credenciais

Aviso de Segurança: Nunca ative a permissão de "Saques" (Withdrawals) na sua API da Binance. O bot precisa apenas de permissão de Leitura e Spot Trading.

Faça cópias dos exemplos fornecidos:

  • Copie user.cfg.example para user.cfg
  • Copie supported_coin_list.example para supported_coin_list.txt

Abra o user.cfg e insira suas chaves da Binance e a GOOGLE_API_KEY (Gerada no Google AI Studio).

3. Lista de Moedas (Watchlist)

Abra o supported_coin_list.txt. Adicione o símbolo das moedas (uma por linha), sem o par USDT. Exemplo:

BTC
SOL
LINK
PEPE

4. Iniciando o Painel UI

Inicie o Dashboard gráfico:

python painel.py

No painel, clique em RUN > Iniciar Bot. Você tem à disposição o placar em tempo real, os gráficos de 15m e o botão 🚨 Venda Forçada para intervenções manuais de segurança.

5. Troubleshooting Comum

  • Erro -1021 (Timestamp outside recvWindow): Relógio do Windows atrasado. Vá em Data/Hora e clique em "Sincronizar Agora".
  • Erro de Conflito websockets: Rode pip install websockets==11.0.3 para forçar o downgrade exigido pela Binance.

Arquitetura do Sistema (v3.3.0)

A arquitetura evoluiu para um ecossistema Híbrido: Cálculo Local (Pandas-TA) + Validação em Nuvem (LLM).

1. Classe Principal: Strategy (profit_gain_strategy.py)

O motor gerencia a matemática pesada com a seguinte árvore de decisão tridimensional:

  • 4H (Macro): Verifica colapsos semanais via RSI.
  • 1H (Swing): Mede o distanciamento da EMA 21 (Reversão à Média) e calcula o True Bottom exigindo uma queda maior que 2x o ATR.
  • 15m (Gatilho): Cruza o MACD Histogram com anomalias de volume institucional para confirmar a formação da "barriga" da curva de preço.

2. Agente Institucional: MarketAnalyzer (ai_agent.py)

Integrado com gemini-2.5-flash-lite. A IA processa o *Chain of Thought* sob leis estritas:

  • A Lei da Curva: Veta matematicamente ativos onde o macd_histograma_15m_positivo seja FALSE.
  • Trava de Confiança: Impossibilita *scores* >= 90 sem que o Python confirme *Spikes* de volume.

3. Manual Override e IPC (Inter-Process Communication)

A separação entre a Interface (Tkinter) e o Motor (Processo Independente) exige um IPC sem bloqueios (Non-blocking):

  • Ações de UI, como o Panic Button ou adição de novas chances, geram *ghost flags* (arquivos .flag).
  • O loop assíncrono do bot consome essas flags no intervalo de varredura, executando funções como _execute_forced_sell(), liquidando a operação a mercado e assumindo um silêncio tático de 60 segundos automaticamente.
