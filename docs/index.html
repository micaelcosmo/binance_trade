Binance Trade Bot Pro 🤖 AI Edition

Terminal quantitativo automatizado com análise tridimensional profunda (Contexto 12h), trailing stop e gestão institucional em lote validada pelo Google Gemini.

v3.6.0 Python 3.9+ Tkinter UI Gemini 2.5 Flash-Lite

🧠 Visão Tridimensional e Projeção

O motor avalia o mercado fornecendo à IA o histórico completo de Price Action das últimas 12 horas (Macro), EMA curta e MACD. A IA foca exclusivamente em projetar se a estrutura de preço tem força para buscar o alvo de lucro sem alucinar em regras matemáticas.

📈 Risco/Retorno Elástico

Stop Loss dinâmico calculado via ATR (Average True Range) blindado de forma elástica até -7.00% (Altcoins) e -10.50% (BTC). O Trailing Stop persegue alvos invisíveis de +1.50%, suportando a volatilidade do mercado.

🎛️ Controle Dinâmico & Caixa Branca

Ajuste o Desvio Padrão do Bollinger (BB STD) e o tempo de Scan em tempo real direto pela UI. Consulte o "Dossiê do Motor" para uma auditoria matemática completa e transparente, sem depender de logs difíceis de ler.

🚨 Manual Override & Auto-Update

Assuma o controle via Botão de Venda Forçada e atualize o código com segurança (One-Click Update). O Tribunal de Swap da IA julga matematicamente se vale a pena migrar de uma operação presa para uma nova oportunidade perfeita.

Bem-vindo ao Bot

Este manual guiará você desde a instalação até a sua primeira operação automatizada de nível institucional.

1. Instalação e Preparação

Clone o repositório, crie um ambiente virtual e instale as dependências. O projeto utiliza o SDK oficial google-genai nativo para alta performance.

git clone https://github.com/micaelcosmo/binance_trade.git
cd binance_trade
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt

2. Configurando suas Credenciais

Aviso de Segurança: Nunca ative a permissão de "Saques" (Withdrawals) na sua API da Binance. O bot precisa apenas de permissão de Leitura e Spot Trading.

Faça cópias dos arquivos de exemplo fornecidos na raiz do projeto:

  • Copie user.cfg.example para user.cfg
  • Copie supported_coin_list.example para supported_coin_list.txt

Abra o user.cfg com um editor de texto e insira suas chaves da Binance e a sua GOOGLE_API_KEY (Gerada gratuitamente no Google AI Studio).

3. Lista de Moedas (Watchlist)

Abra o arquivo supported_coin_list.txt. Adicione o símbolo das moedas que você deseja que o bot monitore (uma por linha), sem o par USDT. Exemplo:

BTC
SOL
LINK
PEPE

4. Iniciando o Painel UI e Atualizações

Inicie o Dashboard gráfico executando o seguinte comando no terminal:

python painel.py

No painel, utilize os novos seletores para ajustar o tempo de varredura (Scan) e a agressividade do Bollinger (BB STD). Depois, clique em RUN > Iniciar Bot. Você tem à disposição o placar em tempo real, Dossiê de Inteligência e ferramentas de intervenção manual.

Para se manter atualizado com novas versões de estratégia, basta clicar no botão 🔄 Atualizar Versão na interface. O bot fará o download e reiniciará sozinho com segurança.

5. Troubleshooting Comum

  • Erro -1021 (Timestamp outside recvWindow): Relógio do Windows atrasado. Vá em configurações de Data/Hora e clique em "Sincronizar Agora".
  • Erro de Conflito websockets: Em sistemas antigos, pode ser necessário rodar pip install websockets==11.0.3 para forçar o downgrade exigido pela API da Binance.

Arquitetura do Sistema (v3.6.0)

A arquitetura evoluiu para um ecossistema Híbrido padronizado: Cálculo Local Estrito via Dataframes (Pandas-TA) atuando como Porteiro, mais Auditoria Estrutural em Nuvem Generativa (Google Gemini).

1. Estrutura Quantitativa (profit_gain_strategy.py)

O motor (Strategy) centraliza a extração assíncrona dos candlesticks (OHLCV) e atua como a primeira barreira lógica rígida:

  • O Porteiro Python: O algoritmo nativo exige matematicamente que o ativo apresente queda profunda (ATR) e fure a banda inferior de Bollinger (Radar Retroativo de 15m e 1H). Ativos que não atendem à exaustão não são enviados para a IA, economizando custo e processamento.
  • Contexto Macro (12h): O motor injeta no payload JSON o histórico contínuo das últimas 12 horas de Price Action, além da distância da EMA e inclinação direcional do RSI.
  • Gatilho Micro (15m): Confirmação local de anomalias de volume institucional no time-frame de aceleração cruzada ao MACD Histogram.

2. Agente Institucional: MarketAnalyzer (ai_agent.py)

Integrado via SDK nativo com gemini-2.5-flash-lite. A IA recebe o Dossiê já filtrado pelo Porteiro Python e atua focada em projeção:

  • A Lei da Estrutura e Projeção: A IA é instruída a cruzar o momentum atual e o Price Action recente com o único objetivo de prever se o ativo detém liquidez e força para buscar um Take Profit de 2.00% nas próximas 14 horas. Ativos em formato "faca caindo" são vetados sumariamente.

3. Auto-Updater & IPC (Inter-Process Communication)

A separação entre a Interface (Tkinter/Main Thread) e o Motor Quantitativo (Sub-processo Independente via `subprocess.Popen`) exige um IPC robusto (Non-blocking):

  • Safe Update Cycle: O componente perform_update no painel aciona a sincronização remota (`git reset` e `git pull`). Utiliza a leitura de diretório absoluto para o recarregamento na memória, garantindo imunidade a *paths* corrompidos de atalhos do Windows.
  • Ghost Flags: Ações de UI, como ajustes de Bollinger, Scan ou botões de Venda Forçada, geram arquivos .flag que são interceptados e apagados silenciosamente pelo motor durante sua varredura no método _check_ui_flags, permitindo atualização de estado em tempo real.
