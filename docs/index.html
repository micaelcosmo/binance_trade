Binance Trade Bot Pro 🤖 AI Edition

Terminal quantitativo automatizado com análise técnica, trailing stop dinâmico e gestão de risco institucional em lote validada pelo Google Gemini.

v3.1.0 Python 3.9+ Tkinter UI Gemini 2.5 Flash-Lite

🧠 Comitê IA (Get Smart Once)

Integração com Gemini 2.5 Flash-Lite via SDK nativo. O motor Python calcula EMAs e RSI (1H) de todas as moedas e submete um único Dossiê Quantitativo para a IA escolher a campeã do lote.

📈 Trailing Stop Dinâmico

Adeus Take Profit fixo. O motor detecta o lucro, arma um gatilho invisível e persegue a alta da moeda. Só vende quando o ativo perde sua força de tendência.

🛡️ Cooldown de Proteção

Se a IA vetar o mercado com regras Anti-FOMO ou Anti-Faca Caindo, o bot entra em silêncio de rede por 30 minutos. Economia brutal de tokens e segurança contra falsos rompimentos.

👑 Regra de Ouro (Auto-Switch)

Trava de capital preso. Se a moeda lateralizar por mais de 1 hora, o bot compila um novo dossiê de Swap, pede aval institucional da IA e migra o saldo automaticamente.

Bem-vindo ao Bot

Este manual guiará você desde a instalação até a sua primeira operação automatizada.

1. Instalação e Preparação

Clone o repositório, crie um ambiente virtual e instale as dependências. O projeto não usa mais LangChain, garantindo uma instalação leve via google-genai nativo.

git clone https://github.com/micaelcosmo/binance_trade.git
cd binance_trade
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt

2. Configurando suas Credenciais

Aviso de Segurança: Nunca ative a permissão de "Saques" (Withdrawals) na sua API da Binance. O bot precisa apenas de permissão de Leitura e Spot Trading.

Você precisa de dois arquivos cruciais para o bot rodar. Faça cópias dos exemplos fornecidos:

  • Copie user.cfg.example para user.cfg
  • Copie supported_coin_list.example para supported_coin_list.txt

Abra o user.cfg e insira:

  • BINANCE_API_KEY e BINANCE_SECRET: Geradas na sua conta Binance.
  • GOOGLE_API_KEY: Gerada no Google AI Studio. Sem ela, a IA fica em bypass (cega).

3. Lista de Moedas (Watchlist)

Abra o arquivo supported_coin_list.txt. Adicione o símbolo das moedas que deseja que a IA avalie no Dossiê Quantitativo, uma por linha, sem o par USDT. Exemplo:

BTC
CFG
SOL
POL

4. Iniciando o Painel UI

Com tudo configurado, inicie o Dashboard gráfico:

python painel.py

No painel, clique em RUN > Iniciar Bot. Você pode acompanhar a auditoria de tokens consumidos pela IA direto nos logs. Para reiniciar a contagem estatística, utilize o botão 0️⃣ Zerar Placar.

5. Troubleshooting Comum

  • Erro -1021 (Timestamp outside recvWindow): O relógio do Windows está atrasado. Vá nas configurações de Data/Hora do Windows e clique em "Sincronizar Agora".
  • Erro de Conflito websockets: Se a API da Binance acusar conflito de pacote de rede, rode pip install websockets==11.0.3 para forçar o downgrade seguro exigido pela corretora.

Arquitetura do Sistema (v3.1.0)

O Binance Trade Bot Pro atingiu um nível Felipe Faria de engenharia. Ele abandonou a verificação estática de múltiplos requests por minuto e adotou o conceito de Batch Analysis (Análise em Lote).

1. Classe Principal: Strategy (profit_gain_strategy.py)

O motor de execução calcula a matemática pesada usando pandas-ta para poupar o processamento da IA. O fluxo atua em dois modos:

  • Modo Dossiê (Desposicionado): Extrai velas de 1H, calcula EMA (9/21/50), RSI (14) e distância da máxima das últimas 24h para toda a Watchlist. O array JSON completo é submetido em uma única requisição.
  • Modo Monitoramento (Posicionado): Isola a comunicação com a IA e foca unicamente no processamento de Trailing Stop e Stop Loss, protegendo o patrimônio em quedas bruscas de liquidez.

2. Agente Institucional: MarketAnalyzer (ai_agent.py)

Reescrito utilizando o SDK nativo google-genai apontado para o modelo gemini-2.5-flash-lite, garantindo aderência absoluta à cota diária do Free Tier.

  • Chain of Thought: O prompt obriga a IA a escrever análises individuais de cada moeda recebida antes de gerar o veredito final, eliminando alucinações matemáticas.
  • Filtros Dinâmicos: Implementação estrita de regras Anti-FOMO (vetando topos e RSI sobrecomprado) e Anti-Faca Caindo. A IA possui permissão para caçar suportes em tendência de baixa, mas tem veto absoluto contra quedas livres.
  • API Audit: O módulo extrai os metadados brutos da resposta da API (Input e Output tokens) e envia para o System Logger em tempo real.

3. Gerenciamento Assíncrono e Resiliência

O projeto garante que anomalias não travem as threads (UI vs. Bot):

  • Cooldown Shield: Se a IA vetar o mercado inteiro, a variável ai_cooldown_until pausa a geração de novos dossiês por 30 minutos. Proteção automática contra falsos rompimentos e *Rate Limits*.
  • UI Flags: O reset do placar de Trades é feito através da criação do arquivo fantasma reset_trades.flag pelo painel gráfico (Tkinter), que o motor em background consome e deleta sem travamentos IPC (Inter-Process Communication).
  • Auto-Healing: Se a internet cair, a função _get_balance() blinda o processo. Ao voltar, o endpoint get_my_trades da Binance varre a memória e restaura as variáveis de compra.
