Binance Trade Bot Pro 🤖 AI Edition

Terminal quantitativo automatizado com análise técnica, trailing stop dinâmico e gestão de risco validada por Inteligência Artificial (Google Gemini).

v2.8.0 Python 3.9+ Tkinter UI Gemini 2.5 Flash

🧠 Oráculo de IA

Integração com Gemini 2.5 Flash. O bot submete os últimos 5 candles (5m) para a IA analisar a microestrutura, barrando compras em topos falsos ou exaustão de volume.

📈 Trailing Stop Dinâmico

Adeus Take Profit fixo. O motor detecta o lucro, arma um gatilho invisível e persegue a alta da moeda. Só vende quando o ativo recua sua força de tendência.

🛡️ Auto-Healing de Rede

Armadura contra timeouts. Se a internet cair, o bot congela a operação. Se reiniciar, ele consulta o histórico da corretora para restaurar a memória do trade.

👑 Regra de Ouro (Auto-Switch)

Trava de capital preso. Se a moeda lateralizar por mais de 1 hora, o bot caça oportunidades melhores, pede aval da IA e migra o saldo automaticamente.

Bem-vindo ao Bot

Este manual guiará você desde a instalação até a sua primeira operação automatizada.

1. Instalação e Preparação

Clone o repositório, crie um ambiente virtual e instale as dependências. As bibliotecas foram fixadas para garantir compatibilidade (ex: SQLAlchemy 1.4).

git clone https://github.com/micaelcosmo/binance_trade.git
cd binance_trade
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt

2. Configurando suas Credenciais

Aviso de Segurança: Nunca ative a permissão de "Saques" (Withdrawals) na sua API da Binance. O bot precisa apenas de permissão de Leitura e Spot Trading.

Você precisa de dois arquivos cruciais para o bot rodar. Faça cópias dos exemplos fornecidos:

  • Copie user.cfg.example para user.cfg
  • Copie supported_coin_list.example para supported_coin_list.txt

Abra o user.cfg e insira:

  • BINANCE_API_KEY e BINANCE_SECRET: Geradas na sua conta Binance.
  • GOOGLE_API_KEY: Gerada no Google AI Studio. Sem ela, a IA fica em bypass (cega).

3. Lista de Moedas (Watchlist)

Abra o arquivo supported_coin_list.txt. Adicione o símbolo das moedas que deseja que o bot vigie, uma por linha, sem o par USDT. Exemplo:

BTC
CFG
SOL
PEPE

4. Iniciando o Painel

Com tudo configurado, inicie o Dashboard gráfico:

python painel.py

No painel, clique em RUN > Iniciar Bot. Você verá no log o bot analisando a matemática (EMA/RSI) e as mensagens do Oráculo IA aprovando ou vetando a entrada.

5. Troubleshooting Comum

  • Erro -1021 (Timestamp outside recvWindow): O relógio do Windows está atrasado. Vá nas configurações de Data/Hora do Windows e clique em "Sincronizar Agora".
  • Erro SQLAlchemy ArgumentError: Acontece se o Pip instalou o SQLAlchemy 2.0. Resolva rodando: pip install SQLAlchemy==1.4.52.

Arquitetura do Sistema

O Binance Trade Bot Pro é estruturado em duas camadas principais: O Motor de Execução (Estratégia) e o Agente de Inteligência Artificial.

1. Classe Principal: Strategy (profit_gain_strategy.py)

Coração lógico do bot. Ele gerencia o ciclo de vida (Scout, Monitor, Trade). O fluxo de varredura (scan_market) atua em dois modos independentes:

  • Modo Caça (Desposicionado): Calcula a EMA (9x21) e o RSI (14) no timeframe de 5m. Se for tendência de alta e RSI < 70, envia o dataframe para validação do ai_agent.
  • Modo Monitoramento (Posicionado): Ignora as outras moedas e foca na matemática do Trailing Stop e do Stop Loss. Calcula a peak_profit_percentage e recuo de 0.20%.

2. Agente de IA: MarketAnalyzer (ai_agent.py)

Utiliza o framework LangChain para conectar com o Gemini 2.5 Flash via API v1beta do Google Generative AI.

  • O Payload: Formata os últimos 5 candles (Timestamp, Open, High, Low, Close, Vol) em string e injeta num ChatPromptTemplate contendo persona institucional.
  • Output Parsing: A IA é forçada a responder um JSON puro com as chaves {"recomendacao", "confianca", "motivo"}. O código possui um parser (`replace`) para limpar escapes markdown que a IA ocasionalmente insere.

3. Resiliência: Error State & Auto-Healing

Adicionado na v2.8.0, impede que o bot quebre ou perca o P/L em anomalias de hardware/rede.

  • Network Shield: A função _get_balance() agora possui tratamento de Timeout, retornando -1.0. O scan_market() lê o número negativo, detecta a falha e dá um return antecipado, congelando o estado (memória).
  • Single Source of Truth: Se a variável self.preco_compra_ativo for perdida (ex: bot reiniciado com saldo pendente), a função _recuperar_dados_compra_real() faz um request no endpoint get_my_trades da Binance, filtra por isBuyer=True, extrai o preço e o timestamp exato, e refaz o estado interno (Auto-Healing).
